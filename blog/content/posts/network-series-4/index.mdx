# 응용 계층은 어떤 역할을 하나요?

응용 계층에서는 사용자 측의 요청을 전달하기 위해 통신 대상이 이해할 수 있는 메세지로 변환하고
전송 계층으로 전달하는 역할을 합니다. 응용 계층은 어플리케이션이 동작하는 계층이며, 서비스를 요청하는 측을
**클라이언트**라고 하며, 서비스를 제공하는 측을 **서버**라고 합니다. 클라이언트 측의 어플리케이션에서
서버 측의 어플리케이션과 통신하려면 응용 계층의 **프로토콜**을 사용해야 합니다. 예를 들어 웹 사이트를
볼 때는 HTTP, 파일을 전송할 때는 FTP, 메일을 보낼 때는 SMTP, 메일을 보낼 때는 POP3이라는 프로토콜을
사용하게 됩니다. 또한 컴퓨터나 네트워크 장비에 붙여진 이름을 기반으로 IP 주소를 알아내는 것을 **이름 해석**이라
하며, DNS를 프로토콜로 사용하게 됩니다.

# 이름 해석이란 것은 어떤 것인가요?

네트워크에 접속을 할 때 우리는 **IP 주소**를 필요로 합니다. 웹 페이지를 제공하는 서버의 데이터를 전송받기 위해
해당 서버의 IP 주소를 알아야 하지만, 우리는 `https://www.naver.com` 등과 같은 도메인으로 웹 페이지를 접근할
수 있게 됩니다. DNS란 이러한 도메인을 접속하고자 하는 서버의 IP로 바꿔주는 역할을 합니다. DNS 서버는 전 세계에
흩어져 있고, 모두 계층적으로 연결되어 있기 때문에 많은 양의 도메인과 IP 주소를 저장하고, 맞는 IP 주소를 빠르게
클라이언트에게 알려줄 수 있습니다.

# 도메인에서 www는 무엇을 의미하나요?

WWW는 World Wide Web의 약자로 HTML, URL, HTTP를 사용해서 제공되는 인터넷 리소스를 의미합니다.
따라서 www는 인터넷보다는 작은 개념으로 인터넷의 리소스는 FTP, Telnet, e-mail, internet relay chat까지
포함합니다. HTML은 마크업 언어로 문자나 이미지, 하이퍼링크를 표현할 수 있게 됩니다. HTML의 예시는 아래와 같습니다.

```html
<!DOCTYPE html>
<html>
<head>
  <title>Name of the website</title>
</head>

<body>
  <h1>Heading for the content below</h1>
  <p>Text text text</p>
</body>

</html>
```

# HTTP란 무엇인가요?

웹 사이트를 보기 위해 서버의 80번 포트를 사용하여 HTTP 요청을 보내고, HTTP 응답을 받습니다. 클라이언트의 포트는
임의로 배정됩니다.  HTTP/1.0 버전에서는 하나의 요청을 보낼 때마다 연결을 수립하고 연결을 끊는 과정을 반복해야해서
느렸습니다. HTTP/1.1 버전에서는 **keepalive**이 있어서 HTTP 요청들이 모두 마무리 된 후 연결을 끊습니다.

![image](./http1-1.png)

HTTP/2.0 버전에서는 HTTP/1.1 버전과 다르게 보낸 요청에 대해 순서대로 응답하는 것이 아닌 빨리 처리되는 것부터
응답을 진행합니다. 또한 Header Compression을 사용해서 중복되는 header는 보내지 않는 방식으로 좀 더 빠르게-
처리됩니다.

[HTTP/2.0 알아보기](https://goodgid.github.io/HTTP-2.0/)

HTTP/3.0도 이미 2020.9에 배포되었습니다. 이후에 HTTP 프로토콜만 따로 게시글로 정리해보겠습니다. :)

# 메일 프로토콜인 SMTP, POP3은 무엇인가요?

SMTP는 메일을 보낼 때 사용하는 프로토콜이며, POP3은 메일을 받는데 사용하는 프로토콜입니다.
SMTP는 포트 번호로 25번을 사용하고, POP3은 110번을 사용합니다.

# 요약

**응용 계층**에서는 어플리케이션에서의 통신 프로토콜을 관리하는 역할을 담당합니다. 어플리케이션의 종류에 따라
FTP, SMTP, POP, HTTP 등 다양한 프로토콜이 있으며 이를 통해 서버와 클라이언트 간의 통신을 가능하게 합니다.
